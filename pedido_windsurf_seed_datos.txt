PEDIDO PARA CLAUDE EN WINDSURF — ARCHIVO SEED DE DATOS INICIALES
==================================================================

PROBLEMA CRÍTICO:
Cada vez que se redespliegue en Railway o se migre
la base de datos, se pierden todos los datos de
configuración que el admin cargó manualmente.

SOLUCIÓN:
Crear un archivo seed_data.py que se ejecute
automáticamente al iniciar la app y cargue todos
los datos base si no existen.

==================================================================
CREAR ARCHIVO: seed_data.py
==================================================================
Este archivo debe ejecutarse en app.py al inicio,
DESPUÉS de crear las tablas y ANTES de arrancar Flask.

Debe insertar con INSERT OR IGNORE (nunca sobreescribir):

==================================================================
CÓMO EJECUTAR EL SEED
==================================================================
En app.py, después de init_db() y antes de if __name__:

  from seed_data import cargar_datos_iniciales
  with app.app_context():
      cargar_datos_iniciales()

La función cargar_datos_iniciales() usa INSERT OR IGNORE
para no sobreescribir datos que el admin ya modificó.
